# Google Play Store 登録手順（完全ガイド）

## 全体の流れ

```
Step 1: Google Play デベロッパーアカウント登録（$25・1回のみ）
Step 2: App Bundle (AAB) をビルド
Step 3: Google Play Console でアプリを作成
Step 4: ストア情報を入力
Step 5: コンテンツのレーティング質問票に回答
Step 6: ターゲット・広告・プライバシー設定
Step 7: リリースを作成 → 審査に提出
```

---

## Step 1: Google Play デベロッパーアカウント登録

### 1-1. アカウント作成
1. ブラウザで https://play.google.com/console にアクセス
2. Google アカウントでログイン（なければ作成）
3. 「Google Play デベロッパーアカウントに登録」をクリック
4. **個人 or 組織** → 個人の場合は「個人」を選択

### 1-2. 登録料支払い
- 初回登録料: **$25（約3,750円）** ← 1回のみ！（Appleと違って毎年ではない）
- クレジットカードまたはデビットカードで支払い

### 1-3. 本人確認
- 名前・住所・電話番号を入力
- 本人確認書類の提出が必要な場合あり（パスポートや免許証）
- 承認まで **数日〜1週間** かかることがあります

## Step 2: App Bundle (AAB) をビルド

ターミナルで以下を実行：

```bash
cd /Users/tomohikosaitoh/claude/calendar-app
flutter build appbundle --release
```

成功すると以下のファイルが作成されます：
```
build/app/outputs/bundle/release/app-release.aab
```

> ⚠️ Google Play は AAB 形式のみ受け付けます（APKは不可）

### 署名キーの作成（初回のみ）

Google Play にアップロードする AAB は署名が必要です。
Flutter の `--release` ビルドにはデフォルトの debug キーが使われますが、
本番リリースには専用の署名キーが必要です：

```bash
# 1. 署名キー（keystore）を作成
keytool -genkey -v -keystore ~/upload-keystore.jks -keyalg RSA \
  -keysize 2048 -validity 10000 -alias upload

# 2. 質問に答える
#    パスワード: 覚えやすいものを設定（忘れるとアップデート不可！）
#    名前、組織名など
```

次に、署名設定ファイルを作成：

```bash
# android/key.properties を作成（gitにコミットしないこと！）
```

`android/key.properties` の内容：
```
storePassword=あなたのパスワード
keyPassword=あなたのパスワード
keyAlias=upload
storeFile=/Users/あなたのユーザー名/upload-keystore.jks
```

`android/app/build.gradle` に署名設定を追加（`android {` ブロック内）：

```gradle
signingConfigs {
    release {
        keyAlias keystoreProperties['keyAlias']
        keyPassword keystoreProperties['keyPassword']
        storeFile keystoreProperties['storeFile'] ? file(keystoreProperties['storeFile']) : null
        storePassword keystoreProperties['storePassword']
    }
}
buildTypes {
    release {
        signingConfig signingConfigs.release
    }
}
```

その後、再度ビルド：
```bash
flutter build appbundle --release
```

## Step 3: Google Play Console でアプリを作成

### 3-1. 新しいアプリを作成
1. https://play.google.com/console にログイン
2. 「**すべてのアプリ**」→「**アプリを作成**」をクリック
3. 以下を入力：

| 項目 | 入力内容 |
|------|---------|
| アプリ名 | `吉日カレンダー` |
| デフォルトの言語 | `日本語 – ja` |
| アプリまたはゲーム | `アプリ` |
| 無料または有料 | `無料` |

4. 宣言事項にチェックを入れて「**アプリを作成**」

---

## Step 4: ストア情報を入力

左メニュー → 「**拡大** > **ストアの掲載情報**」

### 4-1. メインのストア掲載情報

| 項目 | 入力内容 |
|------|---------|
| アプリ名 | `吉日カレンダー` |
| 短い説明 | `六曜・吉日・祝日がわかる暦カレンダーアプリ。天赦日や一粒万倍日もひと目で確認できます。` |
| 詳しい説明 | `docs/store-listing.md` の Google Play 詳しい説明をコピペ |

### 4-2. 画像素材

| 項目 | サイズ | 説明 |
|------|--------|------|
| アプリアイコン | 512 x 512 px | 1024x1024のアイコンを512にリサイズしてアップロード |
| フィーチャーグラフィック | 1024 x 500 px | アプリの宣伝画像（必須） |
| スクリーンショット | 最低2枚 | 9:16比率推奨（1080 x 1920 px など） |

#### フィーチャーグラフィックの作り方
（Canva などの無料ツールで作成可能）
- サイズ: 1024 x 500 px
- アプリ名「吉日カレンダー」を大きく表示
- アプリのスクリーンショットを配置
- オレンジ系の背景色

#### スクリーンショットの撮影
```bash
# Android エミュレーターでアプリを起動
flutter emulators --launch Pixel_7_Pro_API_35
flutter run -d emulator-5554

# エミュレーターの「カメラ」ボタンでスクリーンショット保存
# または⌘+Shift+P
```

### 4-3. カテゴリ

| 項目 | 選択 |
|------|------|
| アプリのカテゴリ | `ツール` |

## Step 5: コンテンツのレーティング質問票に回答

左メニュー → 「**ポリシー** > **アプリのコンテンツ**」→「**コンテンツのレーティング**」

### 質問票の回答

1. 「**開始**」をクリック
2. メールアドレスを入力
3. カテゴリ: 「**ユーティリティ**」を選択
4. 各質問に回答：

| 質問 | 回答 |
|------|------|
| 暴力的な内容を含みますか？ | いいえ |
| 性的な内容を含みますか？ | いいえ |
| 言葉遣いは不適切ですか？ | いいえ |
| 規制対象物質（薬物等）？ | いいえ |
| ギャンブルを含みますか？ | いいえ |
| ユーザー同士の交流機能？ | いいえ |
| ユーザーの位置情報を共有？ | いいえ |
| 未成年者に不適切な広告？ | いいえ |

5. 「**保存**」→ レーティングが「全ユーザー対象」になることを確認

## Step 6: ターゲット・広告・プライバシー設定

### 6-1. ターゲットユーザー
左メニュー → 「**ポリシー** > **アプリのコンテンツ**」→「**ターゲット ユーザー**」

| 質問 | 回答 |
|------|------|
| ターゲットの年齢層 | 「18歳以上」（子供向けでないため） |

### 6-2. 広告の宣言
左メニュー → 「**ポリシー** > **アプリのコンテンツ**」→「**広告**」

| 質問 | 回答 |
|------|------|
| アプリに広告が含まれますか？ | **はい**（AdMobを有効化したため） |

### 6-3. データセーフティ
左メニュー → 「**ポリシー** > **アプリのコンテンツ**」→「**データ セーフティ**」

重要な質問と回答：

| 質問 | 回答 |
|------|------|
| ユーザーデータを収集または共有しますか？ | はい（広告SDKが収集するため） |
| 位置情報 | おおよその位置情報（広告SDK） |
| 端末ID | 広告ID（広告SDK） |
| アプリの利用状況データ | はい（広告SDK） |
| データは暗号化されていますか？ | はい |
| ユーザーがデータ削除をリクエストできますか？ | はい（アプリ削除で全データ消去） |

### 6-4. プライバシーポリシー
左メニュー → 「**ポリシー** > **アプリのコンテンツ**」→「**プライバシーポリシー**」

URL を入力：
```
https://tomo-aidev.github.io/calendar-app/privacy-policy.html
```

## Step 7: リリースを作成 → 審査に提出

### 7-1. テスト（内部テスト推奨）

初めてのリリースでは、まず内部テストで動作確認することを推奨：

1. 左メニュー → 「**テスト** > **内部テスト**」
2. 「**新しいリリースを作成**」をクリック
3. 「**アプリの署名**」→ 「Google Play アプリ署名を使用」を選択（推奨）
4. `.aab` ファイルをドラッグ＆ドロップ
5. リリース名: `1.0.0`
6. リリースノート:
   ```
   初回リリース
   - 六曜・吉日・祝日がわかるカレンダー
   - 今日の運勢表示
   - 記念日・誕生日管理
   - Googleカレンダー連携
   ```
7. 「**レビュー用に保存**」→「**内部テストとして公開を開始**」

### 7-2. 本番リリース

内部テストで問題なければ：

1. 左メニュー → 「**製品版**」
2. 「**新しいリリースを作成**」
3. 内部テストのリリースを昇格、または `.aab` を再アップロード
4. リリースノートを入力
5. 「**製品版として公開を開始**」

## 審査について

| 項目 | 内容 |
|------|------|
| 審査期間 | 初回は **数時間〜7日**（通常は1-3日） |
| 結果 | メールで通知される |
| 承認された場合 | Google Play で公開 |
| リジェクトされた場合 | 理由が記載されるので修正して再提出 |

### よくあるリジェクト理由

| 理由 | 対策 |
|------|------|
| プライバシーポリシーがない | GitHub Pages のURLを必ず設定 |
| スクリーンショットが不十分 | 最低2枚、アプリの主要機能が分かる画像 |
| データセーフティが未入力 | Step 6-3 を漏れなく入力 |
| コンテンツレーティング未回答 | Step 5 を必ず完了 |
| 広告の宣言漏れ | AdMob使用なら「はい」を選択 |

## 補足: Android 署名キーについて

⚠️ **重要**: 署名キー（keystore）は絶対に失くさないでください！
- キーを失くすとアプリのアップデートが永久に不可能になります
- パスワードも忘れないよう安全に保管してください
- `android/key.properties` は `.gitignore` に追加してリポジトリにコミットしないこと

```
# .gitignore に追加
android/key.properties
*.jks
*.keystore
```
